Notes from reading:

  • The CSS3 transform property lets you lets you translate, rotate, scale, and/or skew any element on the page.
  • The value of the transform property is one or more transform functions (separated by spaces) that will be applied in the order they're provided.
  • Translation functions allow you to move elements left, right, up, or down.
  • A translated element can only be moved relative to its current position.
  • Older browsers will require -webkit-(iOS8, Andorid 4.4.3, BB10) or -ms-(IE9 Only)
  • Transforms don't work on inline elements.
  • The scale(x,y) function scales an element by the defined factors horizontally then vertically. If only one value is provided, it will be used for both the x and y values,
  • Note that you shouldn't declare a new transform: because of the cascade, a second transform would override the first. To declare multiple transformations, provide a space-separated list of transform functions.
  • The rotate() function rotates an element around the point of origin by a specified angle value. As with scale, by default the point of origin is the element's center.
  • Remember that transforms are applied in the order provided.
  • The skew(x,y) function specifies a skew along the x and y axes.
  • You can control the origin from which your transforms are applied. This is done using the transform-origin property.
  • If you want to rotate an element in Internet Explorer, go to http://www.useragentman.com/IETransformsTranslator/ for cross-browser code for a given transform.
  • Transitions allow the values of CSS properties to change over time, essentially providing simple animations.
  • The transition-property property defines the CSS properties of the element that should be transitioned, with all for all properties being the default.
  • You can provide any number of CSS properties to the transition-property declaration, separated by commas. Alternatively, you can use the keyword all to indicate that every supported property should be animated as it transitions
  • Include the -webkit- prefix for all the transition properties, not just the transition-property property.
  • If you want to transition more than one property, but not all, comma-separate them
  • The transition-duration property sets how long the transition will take: the duration of time it takes to go from the default state to the transitioned state.
  • 200ms is generally considered the optimum time for a transition: anything slower will make the website seem slow, drawing generally unwanted attention to what was supposed to be a subtle effect. Anything faster may be too subtle.
  • The transition-timing-function lets you control the pace of the transition in even more granular detail. Do you want your animation to start off slow and become faster, start off fast and end slower, advance at an even keel, or some other variation?
  • You can also describe your timing function more precisely by defining your own cubic-bezier function.
  • Use this website to help with cubic-bezier
  • You can divide the transition over equidistant steps. With the steps function, you define the number of steps and the direction of either start or end,
  • Finally, by using the transition-delay property, it's possible to introduce a delay before the transition begins.
  • The transition property is shorthand for the four transition properties just described.
  • Note that the properties in the shorthand syntax can be in any order, however, if a delay is included, you must also include a duration, and the duration must precede the delay.
  • CSS animations, unlike transitions, allow you to control each step of an animation via keyframes.
  • A keyframe is a snapshot that defines a starting or end point of any smooth transition.
  • Generally, it is best to use CSS for simple-state changes in a mobile environment
  • To animate an element in CSS, you first create a named animation, then attach it to an element in that element's property declaration block.
  • Inside each keyframe include the properties you want to animate, along with the animated values. The values will be smoothly interpolated by the browser's animation engine between each keyframe.

  • Information leakage can occur if scripts from one domain can access information (e.g. read pixels) from images from another domain (this is called a cross-origin request). To mitigate this, canvas elements are defined with a flag indicating whether they are origin-clean.
  • A major downside of canvas in its current form is its lack of accessibility. The canvas doesn't create a DOM node, is not a text-based format, and is thus essentially invisible to tools such as screen readers.
  • SVG stands for Scalable Vector Graphics,
  • XML stands for eXtensible Markup Language. Like HTML, it's a markup metalanguage. In plain English, it's a system meant to annotate text. Just as we can use HTML tags to wrap our content and give it meaning, so can XML tags be used to describe data, such as the content of a file.
  • Unlike canvas, images created with SVG are available via the DOM. This enables technologies such as screen readers to see what's present in an SVG object through its DOM node, as well as allowing you to inspect SVG using your browser's developer tools.
  • The viewBox attribute defines the starting location, width, and height of the SVG image.
  • Use for public domain images
  • Drag and Drop API has no mobile support via Android or iOS.
