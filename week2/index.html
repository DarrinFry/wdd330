Notes from reading:

  • ~~~~~~PROGRAMING BASICS~~~~~~
  • RESERVED WORDS:abstract, await, boolean, break, byte, case, catch, char, class, const, continue, debugger, default, delete, do, double, else, enum, export, extends, false, final, finally, float, for, function, goto, if, implements, import, in instanceof, int, interface, let, long, native, new, null, package, private, protected, public, return, short, static, super, switch, synchronized, this, throw, throws, transient, true, try, typeof, var, volatile, void, while, with, yield
  • Not reserved, but you still shouldn't use for variables etc.: undefined, NaN, Infinity.
  • Primitive Data Types: String, Symbol, Number, Boolean, Undefined, Null, and Objects
  • Variable declaration "const" is used when the variable will not ne reassigned to another value.
  • Variable declaration "let" is used if the variable might be reassigned later in the program.
  • If the variable references a non-primitive data type, such as an array, function or object, then using const will not make it immutable. This means the underlying data inside the object can change (known as mutating the object)
  • The variable declaration of 'var" was used in older versions. However, you should use "let" now.
  • ES6 standard you can use blocks { } anything inside the block has local scope.
  • Constant and variable names are case sensitive.
  • A useful way to distinguish between a property and a method ― methods end with parentheses.
  • Template Literals: use the backtick character, `, to deliminate the string and insert variables ${ variable }
  • Symbols: can be used to create unique values, which helps to avoid any naming collisions. The main use-case for symbols is as object property keys. Study this for more info: Sitepoint Article on Symbols
  • If a number starts with a 0x, it is considered to be in hexadecimal (base 16) notation
  • If a number starts with a zero, followed by the letter o, then it is considered to be in octal (base 8) notation
  • If a number starts with a zero, followed by the letter b then it is considered to be in binary (base 2) notation:
  • Infinity is a special error value in JavaScript that is used to represent any number that is too big for the language to deal with. The biggest number that JavaScript can handle is 1.7976931348623157e+308
  • Only 9 values are always false and these are known as falsy values: * "" // double quoted empty string literal * '' // single quoted empty string literal * `` // empty template literal * 0 * -0 // considered different to 0 by JavaScript! * NaN * false * null * undefined
  • ~~~~~ARRAY, LOGIC, and LOOPS~~~~~~
  • Preferred method of creating an array is the array literal " const myArray = []; "
  • Arrays can have a variety of item types inside them.
  • Destructuring an array is the concept of taking values out of an array and presenting them as individual values. Destructuring allows us to assign multiple values at the same time, using arrays: const [x,y] = [1,2];
  • "pop()" removes and returns the last: "shift() removes and returns the first.
  • "push()" adds a new value to the end: "unshift() adds a new value to the beginning.
  • Multidimensional arrays use a coordinate system.
  • set() is a great way to eliminate duplicate values.
  • set() can be converted into an array by using the spread operator or the Array.from() method.
  • Maps can have any kind of data type for key:value
  • To better understand Javascript MAPS I used this video
  • ~~~~~~FUNCTIONS~~~~~~
  • Keep your code DRY (Don't Repeat Yourself)
  • Remember: you need parentheses to invoke a function.
  • If you don't return a value, it will return undefined by default.
  • Any parameters a function needs are set when the function is defined. When a function is invoked, it is provided with arguments.
  • Be aware of rounding errors.
  • The rest operator (...args) can be used to deal with multiple arguments by creating an array of arguments that are available inside the body of the function.
  • Default values should always come after non-default values.
  • Arrow Functions:The parameters come before the arrow and the main body of the function comes after. Arrow functions are always anonymous, so if you want to refer to them, you must assign them to a variable.
  • Arrow Functions: IF the function does not require any parameters, a pair of empty parentheses mus go before the arrow.
  • Functions that are defined using a function declaration are automatically hoisted, meaning they can be invoked before they have been defined.
  • The callback is provided as a parameter, then invoked inside the body of the function.
  • efactoring code ― the functionality of the application remains the same, but the underlying code has become more flexible and easier to maintain, as well as being more readable and descriptive due to the use of functions.
